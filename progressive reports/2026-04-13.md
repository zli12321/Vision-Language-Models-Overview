### Highlights

- **LFM2.5-VL-450M** is a 450M-parameter edge-deployed VLM achieving sub-250ms inference on NVIDIA Jetson Orin. Supports bounding box prediction, multilingual input, and function calling with a 32K-token context window.
- **EXAONE 4.5** scores 77.3 average across 5 STEM benchmarks, outperforming GPT-5-mini (73.5), Claude 4.5 Sonnet (74.6), and Qwen-3 235B (77.0). Supports 6 languages and excels at complex document understanding (contracts, technical drawings, financial statements). Open weights planned on Hugging Face.
- **Gemma 4** is Google's most capable open-weight family, built from Gemini 3 research. The 31B model ranks #3 on Arena AI's open-source text leaderboard. Features 128K–256K context, native function calling, and Apache 2.0 licensing. E2B/E4B variants additionally support audio input.
- **Granite 4.0 3B Vision** targets enterprise document understanding: table extraction, chart comprehension, and key-value pair extraction in a compact 3B footprint.
- **InternVL-U** unifies understanding, reasoning, image generation, and image editing in a single 4B model. Uses CoT reasoning to align user intent with visual generation. Outperforms BAGEL (14B) on generation/editing tasks despite being 3.5× smaller.
- **GLM-4.6V** integrates native function calling for multimodal tool use (process images/screenshots as tool parameters directly). 128K-token context. MIT license; Flash variant offered free via API.

### 3.1 RL Alignment for VLM

> README latest entry: **wDPO — 03/2026**

| Title | Date | Paper | RL Method | Code |
|-------|------|-------|-----------|------|
| Vero: An Open RL Recipe for General Visual Reasoning | 04/2026 | [Paper](https://arxiv.org/html/2604.04917v1) | Task-routed rewards across 6 categories; GRPO-based | [Code](https://github.com/TIGER-AI-Lab/Vero) |

#### Key Insights
- **Vero** provides a fully open VLM family (weights, data, code) trained with RL across 6 visual reasoning task categories. Starting from Qwen2.5-VL-7B and Qwen3-VL-8B, Vero improves by 3.7–5.5 points on average and outperforms Qwen3-VL-8B-Thinking on 23 of 30 benchmarks. A key finding is that different task categories elicit qualitatively distinct reasoning patterns that transfer poorly in isolation, suggesting broad data coverage is the primary driver of RL scaling.

## Key Trends Observed

1. **Open-weight models surge in capability and diversity**: Gemma 4 (Apache 2.0, up to 31B), EXAONE 4.5 (33B, open weights planned), and GLM-4.6V (MIT license) significantly expand the open-weight VLM landscape. Competition at the top is no longer limited to proprietary models.

2. **Edge-deployable VLMs arrive**: LFM2.5-VL-450M and Gemma 4 E2B/E4B specifically target on-device inference. Sub-250ms latency on mobile/edge hardware while retaining bounding box prediction and multilingual support signals a new deployment frontier.

3. **Unified multimodal models go compact**: InternVL-U (4B) and Granite 4.0 3B Vision demonstrate that understanding + generation + editing can coexist in sub-5B parameter models, outperforming much larger baselines on specific tasks.

4. **RL training needs broad data, not just clever algorithms**: Vero's key finding—that narrow task-category RL yields poorly transferable reasoning patterns—highlights data coverage breadth as the dominant factor in RL-based VLM improvement.

5. **Vision token efficiency is a first-class concern**: QAPruner (co-optimizing pruning + quantization), Firebolt-VL (linear-time inference via Liquid decoder), and CoME-VL (entropy-guided encoder fusion) all address the computational cost of processing visual tokens.

6. **Web agents go fully visual**: MolmoWeb's full paper release shows screenshot-only web agents (no HTML/accessibility tree) can match or beat SoM agents on closed models. Combined with open data/code, this democratizes web agent research.

7. **Hallucination mitigation shifts to training-free methods**: Focus Matters proposes inference-time attention intervention without any fine-tuning, complementing the pre-generation detection approach of HALP from the previous report.

8. **VLM "visual blindness" gains formal characterization**: The "VLMs Need Words" paper provides rigorous evidence that VLMs depend on semantic anchors (nameable entities) rather than raw visual detail—identifying this as a training shortcut rather than an architectural ceiling.

9. **Enterprise-specific VLMs emerge**: IBM Granite 4.0 3B Vision is purpose-built for document extraction workflows (tables, charts, KV pairs), signaling growing demand for domain-specialized compact VLMs beyond general-purpose benchmarks.

10. **Autonomous driving VLAs consolidate the MoT paradigm**: UniDriveVLA's Mixture-of-Transformers with decoupled expert pathways continues the trend from HiST-VLA and AutoMoT, establishing expert-decoupled architectures as the dominant design pattern for driving VLAs.
